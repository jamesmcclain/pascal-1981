Three defects, ordered by blast radius. Each step ends with the full suite green.

## Phase 1 — Unify the buffer-variable model with the formatted readers (bug #2, highest priority)

Root cause: `RESET` eagerly fills the FCB buffer (implicit GET), but `pas_fread_int/word/real/char/lstring`, `pas_freadset`, and `pas_fread_filename` read the raw `FILE*`, skipping the buffered component.

Fix strategy — make PENDING the universal post-RESET state, so one mechanism owns "first read":

1. In `pas_file_reset`, replace the eager `raw_get(f, 1)` with `set_mode_flags(f, MODE_READ, 0, 0, /*pending=*/1)`. `force_fill` already converts PENDING → real GET at every `F^`/EOF/EOLN use site, so buffer-variable semantics are preserved (verify: existing `TestFileBufferModel` round-trip tests must still pass unchanged).
2. Add a small helper `static int fcb_next_char(struct pas_file_fcb *f)` that (a) if buffer holds a live unconsumed component (not PENDING, not EOF), returns it and marks it consumed; else (b) `fgetc`. Define "consumed" with one new mode flag (e.g. `MODE_CONSUMED` 0x100) rather than touching the FCB layout.
3. Route `pas_fread_*`, `pas_freadset`, and `pas_fread_filename` through `fcb_next_char`/a matching `fcb_unget`. The `unread_to`/`ungetc` calls become "stash back into the buffer + set PENDING-cleared state," which also kills the current ungetc-vs-buffer divergence.
4. Edge cases to encode in tests: RESET → READ(f, c) yields the *first* char; RESET → READSET yields the full token; GET → READ interleave; EOLN/EOF observed correctly after each reader; stdin path unaffected (lazy attach already sets PENDING).

## Phase 2 — Reject writes in inspection mode (bug #1)

Root cause: `stream_for(f, writing=1)` silently flips MODE_READ → MODE_WRITE at the current offset.

1. In `stream_for`, when `writing` and `current_mode == MODE_READ`, `die("file runtime: WRITE requires REWRITE/write mode")` — consistent with the existing GET/PUT mode aborts. Keep the CLOSED → write auto-open only if a named REWRITE-like default is actually wanted; simpler is to also abort on CLOSED and require REWRITE, matching the manual.
2. Symmetric check: `writing == 0` with `current_mode == MODE_WRITE` should also abort (reading a generation-mode file).
3. When 9.x TRAP/ERRS lands, these `die` sites become the trap dispatch points; leave a comment marking them.
4. Tests: hostile run asserting nonzero exit + stderr message for `RESET; WRITE(f, …)` and `REWRITE; READ(f, …)`; regression test that the file on disk is *not* modified.

## Phase 3 — Close the F.TRAP/F.ERRS typecheck/codegen split (8.6)

Pick the loud option now (cheap, honest): make the typechecker reject `F.TRAP`/`F.ERRS` on file variables with a "not yet implemented" diagnostic, mirroring the codegen error, so failure moves to the earlier phase with a better message. Direct `FCBFQQ` record variables stay as-is. Re-grade the 8.6 note: "source can name them" → "rejected at typecheck pending trapped-I/O work." Lowering real TRAP/ERRS slots is deferred to the trapped-I/O item, where the Phase 2 `die` sites get converted.
